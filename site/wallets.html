How compatibility works

WalletGuard Pro hooks at the window.ethereum provider level, the same standard every EVM wallet implements (EIP-1193). This means:

  • No per-wallet integration required. If a wallet injects window.ethereum, we work with it.
  • No wallet team approval needed. We don't partner with wallet vendors; we work with the user's browser directly.
  • Updates ship immediately to all users. When we add a new heuristic, every WalletGuard user gets it within 24 hours via the Chrome/Firefox auto-update mechanism.

What about non-EVM chains?

WalletGuard Pro is currently EVM-only. This covers ~80% of Web3 wallet users by activity. Non-EVM support (Solana, Bitcoin, Cosmos, Sui, Aptos) is on the roadmap but requires separate risk engines for each chain's transaction format.

Found an incompatibility?

Open an issue at github.com/eupho808/walletguard-pro/issues with:

  • Wallet name and version
  • Browser and OS
  • What you expected to happen
  • What actually happened (with screenshot or video if possible)

Don't see your wallet?

If your wallet injects window.ethereum (which 99% of EVM wallets do), WalletGuard works with it. Just install and try.
