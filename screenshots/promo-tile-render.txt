# Re-rendering screenshots — step by step

The popup mock (`popup-mock.html`) and promo tile (`promo-tile.svg`) were
updated to v4 CALM design (no cyberpunk, no neon glow, no decorative
gradients). The four overlay screenshots (`01-`, `02-`, `03-`, `05-`) are
honest captures of the actual extension overlay — they are kept as-is.

You need to re-capture two PNGs.

## 1. `04-approval-scanner.png` (popup view) — manual in Chrome

1. Open `screenshots/popup-mock.html` in Chrome (drag the file onto a
   Chrome window, or `file:///` + path)
2. F12 → Ctrl+Shift+M (device toolbar) → set viewport to **1280 × 800**
3. Ctrl+Shift+P → "Capture screenshot"
4. Save as `screenshots\04-approval-scanner.png`

If Chrome's screenshot tool saves with extra padding or the wrong size,
right-click the rendered popup → "Save image as..." (Chrome can save the
selected element). Otherwise use the manual DevTools flow above.

## 2. `promo-tile.png` (440×280) — automated via Edge headless

The SVG alone does NOT fill the viewport when opened in Chrome directly
(Chrome renders it at native size with body margins). Use the wrapper
HTML file (`_render-promo.html`) plus Edge headless:

```powershell
$edge = "${env:ProgramFiles(x86)}\Microsoft\Edge\Application\msedge.exe"
& $edge --headless --disable-gpu --hide-scrollbars `
  --window-size=440,280 `
  --screenshot="$PWD\screenshots\promo-tile.png" `
  "file:///$PWD\screenshots\_render-promo.html"
```

The wrapper file (`screenshots\_render-promo.html`) loads
`promo-tile.svg` in a `<img>` tag forced to `width: 440px; height: 280px`
on a dark background. Edge captures the exact viewport.

If Edge is not at the path above, try the Chrome variant:

```powershell
$chrome = "${env:ProgramFiles}\Google\Chrome\Application\chrome.exe"
& $chrome --headless --disable-gpu --hide-scrollbars `
  --window-size=440,280 `
  --screenshot="$PWD\screenshots\promo-tile.png" `
  "file:///$PWD\screenshots\_render-promo.html"
```

Both confirmed working on Windows 10/11 with Edge 100+ / Chrome 100+.

## Cleanup

If a misnamed file (`screenshots\screenshots04-approval-scanner.png`)
appears in `git status`, remove it:

```powershell
git rm screenshots/screenshots04-approval-scanner.png
```

## Commit + push

After both PNGs are re-captured:

```powershell
git add screenshots/04-approval-scanner.png screenshots/promo-tile.png
git add screenshots/popup-mock.html screenshots/_render-promo.html
git commit -m "screenshots: capture v4 CALM PNGs (popup + promo tile)"
git push origin main
```

## Verification

After commit:

- `git log --oneline -3` should show the new commit on top
- `screenshots/promo-tile.png` should be exactly 440×280, solid dark
  background with single emerald shield + check, "WalletGuard Pro"
  wordmark (white + emerald "Pro"), tagline, four small-caps meta items
- `screenshots/04-approval-scanner.png` should be 1280×800 with the
  actual extension popup on the left (v4 CALM) and the pitch column on
  the right
- No glow, no neon, no decorative gradients anywhere
