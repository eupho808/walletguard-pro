# WalletGuard Pro v3.7.0 — Project Context

I'm sending you the source ZIP of **WalletGuard Pro v3.7.0 EXPIRY** (commit `6ae6cb6`). Open-source Web3 wallet security extension. MIT licensed, 1,429 automated tests across 33 suites, all green.

## What it is

A Manifest V3 Chrome/Firefox extension that sits between your dApp and your wallet (MetaMask, Rabby, Frame, etc.). It intercepts every transaction, signature, and approval request, decodes the calldata, decodes Permit / Multicall / Universal Router payloads, and shows you the risk with an explicit breakdown before your wallet does. Catches unlimited ERC-20 approvals, NFT `setApprovalForAll`, phishing sites, typosquatted domains, and 26 other attack patterns.

## Architecture

- **`injector.js`** — MAIN world, `document_start`, `<all_urls>`, all_frames. Wraps `window.ethereum` via Proxy. The single most security-critical file.
- **`content.js`** — ISOLATED world, same triggers. Analysis pipeline, risk engine, UI overlay, inlined copies of all 31 lib modules. ~9,800 lines.
- **`background.js`** — Service worker. Message routing, state, approval scanner orchestration, alarms. Also inlines minimal copies of lib helpers because SW can't ES-import popup-bundle modules.
- **`popup.html` / `popup.css` / `popup.js`** — 360px popup UI, v4 CALM aesthetic.
- **`settings.html` / `settings.js`** — full settings page in a new tab.
- **`lib/`** — 31 pure-utility modules (risk engine, decoders, scanners, address book, ENS, price oracle, etc.).
- **`packages/walletguard-core/`** — extracted core lib for npm publishing (future).
